# Aero Taxi Credit Card Manual Testing Project

## Project Overview
This repository contains a manual QA testing project for the Aero Taxi web application. The project focused on validating the credit card payment section, input validation rules, fare calculation, and travel time calculation.

The project was completed as part of QA engineering coursework using manual testing techniques, equivalence partitioning, boundary value analysis, and structured test case design.

## Features Tested

### Credit Card Validation
- Card entry before ordering
- Add button behavior
- Maximum number of cards allowed
- Card number numeric validation
- Card number character limits
- Card number formatting
- Automatic spacing behavior
- Incomplete input validation

### CVV/CVC Validation
- Numeric-only input validation
- Character length validation
- Minimum and maximum accepted values
- Input format validation
- Add button disabled state for incomplete input

### Aero Taxi Calculations
- Travel time calculation
- Fare calculation
- Display of travel time results
- Display of travel cost results

### Security Requirement
- Encryption of payment details during transmission

## QA Techniques Used

- Manual test case design
- Test execution
